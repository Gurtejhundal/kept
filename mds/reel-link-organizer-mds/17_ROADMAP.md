# Delivery Roadmap

## Phase 0 — Validation
- Build clickable mobile prototype
- Test with 8–12 students
- Validate share-sheet understanding
- Measure whether users can complete a save without instruction
- Validate date and visual retrieval

Exit condition:
At least 70% of test users complete the core save flow without assistance.

## Phase 1 — Foundation
- Monorepo setup
- Authentication
- Database schema
- Design tokens
- Basic mobile navigation
- Basic web shell
- Logging and analytics

## Phase 2 — Capture MVP
- Share intent
- Paste/manual save
- URL extraction
- Thumbnail upload
- Save composer
- Background metadata enrichment
- Item detail

## Phase 3 — Library and search
- Timeline home
- Visual cards
- Full-text search
- Date filters
- Categories
- Collections
- Archive and trash

## Phase 4 — Web management
- Desktop library
- Grid/list views
- Bulk edit
- Collection management
- Export
- Share links

## Phase 5 — Reliability and release
- Link normalization
- Duplicate detection
- Reel availability checks
- Notification preferences
- Accessibility audit
- Security audit
- App Store and Play Store release work

## Phase 6 — Premium
- OCR
- Advanced filters
- Semantic search
- Reel deletion alerts
- Unlimited archive
- Smart resurfacing
- Subscription infrastructure

## First-release prioritization
1. Capture
2. Save reliability
3. Visual timeline
4. Date search
5. Text search
6. Collections
7. Sharing
8. Monitoring
9. Premium

Do not reverse this order. A polished sharing system is worthless if saving is still inconvenient.
