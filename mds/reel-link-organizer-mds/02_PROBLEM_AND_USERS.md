# Problem and Users

## Problem statement
Instagram chatbot campaigns commonly work like this:

1. User watches a reel.
2. User comments a keyword such as “link,” “course,” or “dress.”
3. A chatbot sends a DM.
4. The DM contains a destination link.
5. The user does not need the item immediately.
6. Days or weeks later, the user remembers the reel but not the inbox message.
7. The link becomes difficult to recover.

Instagram DMs are chronological conversations, not a structured personal archive. The user often remembers:
- The visual appearance of the reel
- The creator
- The approximate week or month
- A vague keyword
- The product type

The inbox does not reliably support those memory cues.

## Primary persona

### Student discovery-heavy user
- Age: 18–27
- Uses Instagram daily
- Follows creators and recommendation pages
- Saves links for later purchases or research
- Discovers fashion, skincare, tools, courses, internships, study notes, cafes, and travel places
- Has high save intent but weak follow-through
- Prefers visual recall over file-system organization

## Secondary personas
- Young professionals
- Content-heavy shoppers
- Students researching courses and resources
- Friends sharing recommendation lists
- Creators organizing inbound collaboration and inspiration links

## User jobs to be done
- Save a link with its reel context.
- Find something from last week or last month.
- Search by partial memory.
- Browse visually instead of reading old DM threads.
- Group items into collections.
- Share selected collections.
- Know whether the original reel still exists.
- Open the final link without returning to Instagram.

## User frustrations
- Searching Instagram DMs is slow and unreliable.
- Screenshots preserve the visual but not the clickable destination.
- Copying links into notes removes context.
- Generic bookmark apps look like lists of URLs.
- Manual categorization feels like work.
- Users forget which creator sent the link.
